## Key decisions made
- Decision: Gate the existing hover-flip behind `@media (hover: hover)`.
  Why: `:hover` is sticky on touch devices — a tap leaves the card flipped until you tap elsewhere; the media query scopes it to real pointer devices.
- Decision: Drive the mobile flip with delegated, passive touch handlers on `document` that never `preventDefault`.
  Why: Delegation handles cards rendered after the async product fetch (no per-card listeners); passive + no preventDefault keeps vertical scrolling fully intact; `elementFromPoint` on each move naturally flips each image the finger passes over while scrolling.
- Decision: Preview locally with `netlify dev` rather than a plain static server.
  Why: The catalogue/featured pages fetch `/api/get-products`, a Netlify function that pulls from Airtable using `AIRTABLE_API_KEY` from `.env`; a static server 404s that endpoint ("Couldn't load products right now / HTTP 404").
